# Documentation

`codeye` is a git-aware repository metrics CLI focused on speed, predictable output, and low operational overhead.

Use these sections as the production reference:

- Start with [Install](/docs/install/) for Go, direct binary, and hosted installer paths.
- Use [Usage](/docs/usage/) for the main command surface, analysis modes, and subcommands.
- Use [Configuration](/docs/config/) for `.codeye.toml`, environment variables, and precedence rules.
- Use [Output Formats](/docs/formats/) when wiring `codeye` into CI, badges, or downstream scripts.
