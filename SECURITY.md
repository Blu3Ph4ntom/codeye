# Security Policy

## Supported versions

Security fixes are applied to the latest supported code on `main`.

## Reporting a vulnerability

Do not open a public issue for suspected security vulnerabilities.

Report privately through GitHub Security Advisories:

- [Create a private vulnerability report](https://github.com/Blu3Ph4ntom/codeye/security/advisories/new)

Include:

- a clear description of the issue
- affected versions or commits if known
- reproduction steps
- impact assessment

You can expect an initial response within a reasonable best-effort window. Public disclosure should wait until a fix or mitigation is available.
